# 난동구리 테스트 가이드

## 1. 준비

난동구리는 다음 두 부분이 함께 실행되어야 합니다.

1. **난동구리 데스크톱 앱**: 3D 너구리, 효과음, 알림과 답변 이동을 담당합니다.
2. **브라우저 확장 프로그램**: ChatGPT와 Gemini의 답변 완료를 감지합니다.

확장 프로그램을 아직 설치하지 않았다면 [확장 프로그램 설치 가이드](EXTENSION_INSTALL.md)를 먼저 따라 하세요.

## 2. 앱 실행 확인

### macOS

1. 난동구리 앱을 실행합니다.
2. 일반 설정창이 바로 나타나지 않아도 정상입니다.
3. 화면 상단 메뉴바에서 난동구리 아이콘을 찾습니다.
4. 아이콘을 왼쪽 클릭합니다.
5. 작은 설정 패널이 열리는지 확인합니다.
6. 같은 아이콘을 다시 왼쪽 클릭해 패널이 닫히는지 확인합니다.
7. 아이콘을 오른쪽 클릭해 `설정 열기`와 `난동구리 종료`가 보이는지 확인합니다.
8. 설정창 오른쪽 위 `×`를 눌렀을 때 창만 닫히고 메뉴바 아이콘은 남는지 확인합니다.

### Windows

1. 난동구리 앱을 실행합니다.
2. 작업 표시줄 오른쪽 알림 영역에서 난동구리 아이콘을 찾습니다.
3. 보이지 않으면 숨겨진 아이콘 `^` 버튼을 누릅니다.
4. 아이콘을 왼쪽 클릭해 설정 패널을 엽니다.
5. 아이콘을 오른쪽 클릭해 빠른 메뉴를 확인합니다.
6. 설정창 오른쪽 위 `×`를 눌렀을 때 앱이 종료되지 않고 시스템 트레이에 남는지 확인합니다.

## 3. 연결 상태 확인

1. 설정 패널을 엽니다.
2. 패널 위쪽 상태가 `준비됨`인지 확인합니다.
3. `연결 확인 중`이라면 다음을 확인합니다.
   - 난동구리 앱이 실행 중인지 확인합니다.
   - 확장 프로그램이 켜져 있는지 확인합니다.
   - ChatGPT 또는 Gemini 탭을 새로고침합니다.
   - 3~5초 기다린 뒤 패널을 다시 엽니다.

개발자는 터미널에서도 로컬 브리지를 확인할 수 있습니다.

```bash
curl http://127.0.0.1:43119/health
```

정상 결과:

```json
{"ok":true,"app":"Nandong Guri"}
```

## 4. ChatGPT 실제 테스트

1. 난동구리 앱과 확장 프로그램을 실행합니다.
2. ChatGPT 탭을 새로고침합니다.
3. 답변이 조금 길게 나오는 질문을 입력합니다.
4. 답변 생성이 시작되면 다른 탭이나 다른 앱으로 이동합니다.
5. 답변이 끝난 직후 시스템 알림과 최초 효과음이 발생하는지 확인합니다.
6. 작은 3D 너구리가 화면에 나타나는지 확인합니다.
7. 너구리가 짧게 걷거나 뛴 뒤 한자리에서 춤추는지 확인합니다.
8. 춤이 끝난 뒤 다시 조금 이동하는지 확인합니다.
9. 너구리를 클릭합니다.
10. 새 브라우저 창이 생기지 않고 기존 ChatGPT 탭과 브라우저 창이 앞으로 오는지 확인합니다.
11. 완료된 답변이 표시되고 너구리가 사라지는지 확인합니다.

답변 완료 순간 ChatGPT 탭을 계속 보고 있었다면 너구리는 약 5초 뒤 자동으로 사라지는 것이 정상입니다.

## 5. Gemini 실제 테스트

1. Gemini 탭을 새로고침합니다.
2. 긴 답변을 요청합니다.
3. 생성 중 다른 앱으로 이동합니다.
4. 답변 완료 시 너구리가 나타나는지 확인합니다.
5. 너구리를 클릭했을 때 Gemini 답변으로 이동하는지 확인합니다.
6. Gemini 탭으로 직접 돌아갔을 때 너구리가 사라지는지 확인합니다.

## 6. 클릭 통과 테스트

1. 너구리가 나타난 상태에서 너구리가 없는 화면 영역을 클릭합니다.
2. 뒤에 있는 앱이나 버튼이 정상적으로 눌리는지 확인합니다.
3. 마우스를 너구리 위로 옮깁니다.
4. 포인터 커서가 나타나는지 확인합니다.
5. 너구리를 클릭했을 때만 답변 페이지가 열리는지 확인합니다.

너구리가 너무 빨라 클릭하기 어려워서는 안 됩니다. 이동 후 4~7초 동안 제자리 춤을 추는 구간에 쉽게 클릭할 수 있어야 합니다.

## 7. 화면 이동 테스트

1. 너구리가 화면 아래쪽에만 머물지 않는지 확인합니다.
2. 화면 위쪽과 가운데도 이동하는지 확인합니다.
3. 머리, 귀, 발 또는 꼬리가 화면 밖으로 심하게 잘리지 않는지 확인합니다.
4. 걷기 또는 달리기 모션과 화면 이동 방향이 어울리는지 확인합니다.

## 8. 여러 모니터 테스트

1. 모니터를 두 개 이상 연결합니다.
2. 실제 ChatGPT 또는 Gemini 완료 이벤트를 발생시킵니다.
3. 10초 이상 너구리를 클릭하지 않습니다.
4. 너구리가 다음 모니터로 이동하는지 확인합니다.

현재 구현은 하나의 오버레이 창을 다음 모니터로 옮기는 방식입니다.

## 9. 효과음 테스트

1. 설정에서 `효과음`을 켭니다.
2. 답변 완료 시 최초 등장 효과음이 한 번 재생되는지 확인합니다.
3. 너구리가 움직이거나 난동 단계가 상승할 때 소리가 반복되지 않는지 확인합니다.
4. 너구리를 클릭할 때 효과음이 한 번 재생되는지 확인합니다.
5. 효과음을 끈 뒤 같은 과정을 반복해 소리가 나지 않는지 확인합니다.

## 10. 설정 테스트

- 효과음 스위치가 저장되는지 확인합니다.
- 시스템 알림 스위치가 저장되는지 확인합니다.
- 로그인 시 자동 실행을 켜고 끌 수 있는지 확인합니다.
- 너구리 크기 슬라이더가 다음 등장부터 반영되는지 확인합니다.
- 오른쪽의 소개, 사용 방법, 확장 프로그램 설치, 환경 및 단축키를 클릭했을 때 상세 시트가 열리는지 확인합니다.
- 상세 시트 바깥이나 닫기 버튼을 눌렀을 때 닫히는지 확인합니다.
- 설정창을 닫아도 앱과 브라우저 감지가 계속되는지 확인합니다.
- 설정 화면의 장식 너구리들이 제목, 설정명, 스위치, 단축키와 겹치지 않는지 확인합니다.

## 11. 긴급 퇴장 테스트

1. 너구리가 화면에 나타난 상태에서 `Command/Ctrl + Shift + G`를 누릅니다.
2. 너구리가 즉시 사라지는지 확인합니다.
3. 뒤에 있던 앱을 다시 정상적으로 클릭할 수 있는지 확인합니다.

## 12. 개발 모드 실행

```bash
cd desktop-app
source "$HOME/.cargo/env"
npm run tauri dev
```

터미널을 닫으면 개발 모드 앱도 종료됩니다.

## 13. 문제 해결

### 메뉴바 또는 시스템 트레이 아이콘이 보이지 않음

- 앱이 이미 종료되지 않았는지 확인합니다.
- Windows에서는 숨겨진 아이콘 `^` 목록을 확인합니다.
- 앱을 완전히 종료한 뒤 다시 실행합니다.

### 설정 패널이 `연결 확인 중`에서 바뀌지 않음

- [확장 프로그램 설치 가이드](EXTENSION_INSTALL.md)의 `자주 막히는 부분`을 확인합니다.
- 확장 프로그램과 AI 탭을 모두 새로고침합니다.
- `curl http://127.0.0.1:43119/health`가 성공하는지 확인합니다.

### 너구리가 나타나지 않음

- 확장 프로그램이 켜져 있는지 확인합니다.
- ChatGPT 또는 Gemini 탭을 설치 후 새로고침했는지 확인합니다.
- 답변 생성이 실제로 끝날 때까지 기다립니다.

### 너구리를 클릭할 수 없음

- 제자리에서 춤추는 구간에 다시 클릭합니다.
- 긴급 퇴장 단축키로 오버레이를 닫은 뒤 앱을 다시 실행합니다.

### 효과음이 들리지 않음

- 설정의 효과음을 켭니다.
- 시스템 음량, 음소거와 출력 장치를 확인합니다.

## 14. 현재 제한

- ChatGPT와 Gemini 웹 UI가 변경되면 완료 감지 코드 수정이 필요할 수 있습니다.
- Safari와 Firefox는 아직 지원하지 않습니다.
- Windows 설치 파일은 Windows 또는 CI 환경에서 별도로 빌드해야 합니다.
- macOS 배포 파일은 빌드한 CPU 아키텍처와 일치해야 합니다.
